# DofeAI 脚手架说明

本仓库作为 **DofeAI Monorepo 脚手架**，用于基于该模板创建新项目或作为现有项目的参考结构。

## 一、脚手架范围：保留除日志外的全部文件

- **保留**：除「日志」外的所有源码、配置、文档、脚本、迁移等，均纳入脚手架并进入版本库。
- **排除**：仅 **日志相关** 不进入版本库，运行或构建时产生的日志目录与日志文件一律不提交。

## 二、排除项（仅日志）

以下内容已通过根目录 `.gitignore` 排除，**不会**出现在脚手架中：

| 排除项             | 说明                               |
| ------------------ | ---------------------------------- |
| `logs/`            | 任意层级的日志目录                 |
| `**/logs/`         | 同上，递归匹配                     |
| `*.log`            | 所有 `.log` 文件                   |
| `npm-debug.log*`   | npm 调试日志                       |
| `yarn-debug.log*`  | yarn 调试日志                      |
| `yarn-error.log*`  | yarn 错误日志                      |
| `pnpm-debug.log*`  | pnpm 调试日志                      |
| `lerna-debug.log*` | lerna 调试日志                     |
| `*.log.*`          | 带后缀的日志文件（如按日切割）     |
| `*.gz`             | 常见日志压缩格式（与上述规则配合） |

除以上日志相关项外，`.gitignore` 中其余规则（如 `node_modules/`、`dist/`、`.env` 等）为常规开发/构建忽略项，与「脚手架保留范围」无关。

## 三、使用方式

### 1. 通过 create-dofe-ai 生成项目脚手架（推荐）

**若包已发布到 npm**，在任意目录下执行：

```bash
npx create-dofe-ai <项目目录名>
# 或
pnpm create dofe-ai <项目目录名>
```

**若包尚未发布**（例如你 clone 了本仓库），在**仓库根目录**执行：

```bash
pnpm run export-scaffold              # 首次需生成模板（仅一次）
pnpm run create-scaffold -- my-app    # 生成到当前目录下的 my-app
```

示例（生成后进入项目并启动）：

```bash
npx create-dofe-ai my-app   # 或 pnpm run create-scaffold -- my-app
cd my-app
pnpm install
pnpm db:generate   # 若使用 API + Prisma
pnpm dev
```

- 生成的目录包含当前仓库 **git 中跟踪的几乎全部文件**（源码、配置、文档、脚本、迁移等），仅排除 `packages/create-dofe-ai/` 自身。
- 若需交互式配置端口、数据库、Redis、作者等，可在生成的项目中执行：`node scripts/init-project.js`。

**关于 404**：若执行 `npx create-dofe-ai my-app` 出现 `404 Not Found`，说明包尚未发布到 npm，请改用上述「若包尚未发布」的方式（clone 本仓库后使用 `pnpm run create-scaffold -- my-app`）。

**发布到 npm**：完整步骤见 [发布到 npm](发布到npm.md)。简要流程：在根目录执行 `pnpm run export-scaffold`，再在 `packages/create-dofe-ai` 下执行 `npm publish`（若为 scope 包需加 `--access public`）。

### 2. clone 本仓库

```bash
git clone <repository-url> my-project
cd my-project
```

克隆所得即「除日志外全部文件」的脚手架，不会包含任何日志文件。

### 3. 基于模板创建新仓库

使用 GitHub/GitLab「Use this template」或本地拷贝仓库时，仅会包含已提交内容，日志不会纳入。

### 4. 本地开发

运行 `pnpm dev` 等命令时，若产生 `logs/` 或 `*.log`，它们会落在本地且被 `.gitignore` 忽略，不会被提交回脚手架。

## 四、如何确认「仅排除日志」

- 查看根目录 `.gitignore` 中 **「# Logs」** 段，所列规则即脚手架中唯一定义「不保留」的内容。
- 若需打包或分发脚手架（如 `git archive`、拷贝目录），默认不会包含被 git 忽略的文件，因此打包结果中不会带有 `logs/` 或 `*.log`。

## 五、其它说明

- 环境变量、密钥、本地配置等按 `.gitignore` 其它段落处理，与「日志排除」分开。
- 若在 CI 或自动化脚本中「从本仓库生成脚手架产物」，请确保仅排除上述日志规则，不额外排除业务源码或配置，以实现「保留除日志外的全部文件」。

## 六、基础设施依赖（@dofe/infra-\*）

本脚手架的后端基础设施层（`libs/infra/`）已提取为独立仓库 `infra.dofe.ai`，包含 10 个 `@dofe/infra-*` 子包，已发布到 npm。脚手架通过 `package.json` 依赖（如 `^0.1.56`）从 npm 安装 `@dofe/infra-*`，**无需**克隆 `infra.dofe.ai` 或在 `pnpm-workspace.yaml` 中引用同级目录。

**使用脚手架前需确保**：

1. 运行 `pnpm install` 后确认 `node_modules/@dofe/infra-*` 已从 npm 正确安装
2. 如需本地调试 infra 源码，可自行在 `pnpm-workspace.yaml` 中添加对同级 `infra.dofe.ai` 的引用（默认不启用）
