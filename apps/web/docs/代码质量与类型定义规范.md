# 代码质量与类型定义规范

> 本文档为脚手架占位文档，请在业务项目中按需完善。

## 概述

本规范定义了前端代码质量和 TypeScript 类型定义的标准，确保代码的可维护性和类型安全。

---

## 1. 类型定义标准

### 1.1 使用 `import type` 导入类型

```typescript
// ✅ 正确：使用 import type 导入纯类型
import type { User, UserRole } from '@/types/user';

// ❌ 错误：混合导入类型和值
import { User, UserRole, createUser } from '@/types/user';
```

### 1.2 Props 类型定义

```typescript
// ✅ 正确：使用 interface 定义 Props
interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
  children: React.ReactNode;
  onClick?: () => void;
}

function Button({ variant = 'primary', size = 'md', children, onClick }: ButtonProps) {
  // ...
}

// ❌ 错误：使用 React.FC
const Button: React.FC<ButtonProps> = ({ children }) => {
  // ...
};
```

### 1.3 API 响应类型

```typescript
// ✅ 正确：使用 Zod schema 推断类型
import { z } from 'zod';

const UserSchema = z.object({
  id: z.string().uuid(),
  name: z.string(),
  email: z.string().email(),
});

type User = z.infer<typeof UserSchema>;
```

## 2. 代码质量规范

### 2.1 避免 `any` 类型

```typescript
// ✅ 正确：使用具体类型或 unknown
function processData(data: unknown): void {
  if (typeof data === 'string') {
    console.log(data.toUpperCase());
  }
}

// ❌ 错误：使用 any
function processData(data: any): void {
  console.log(data.toUpperCase()); // 运行时可能报错
}
```

### 2.2 使用严格的 null 检查

```typescript
// ✅ 正确：显式处理 null/undefined
function getUser(id: string): User | null {
  const user = users.find(u => u.id === id);
  return user ?? null;
}

// 使用时
const user = getUser('123');
if (user) {
  console.log(user.name);
}
```

---

## 3. 类型检查清单

- [ ] 所有函数参数都有明确的类型定义
- [ ] 所有函数返回值都有明确的类型定义
- [ ] 避免使用 `any`，优先使用 `unknown` + 类型收窄
- [ ] 使用 `import type` 导入纯类型
- [ ] Props 使用 `interface` 定义，不使用 `React.FC`
- [ ] API 响应类型使用 Zod schema 推断
