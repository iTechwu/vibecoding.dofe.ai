## 本轮产品判断

Loops 的产品心智已经从“agent 控制台”转向“AI 工程交付控制面”。当前最重要的 UIUX 原则是：

1. 默认路径只暴露用户能理解的动作，如创建需求、审阅 Spec、继续推进 Loop；
2. 内部调度动作保留为系统证据，不成为普通用户的主任务；
3. 每个被禁用的关键动作都必须给出可见原因；
4. 详情页和 Dashboard 使用同一套“人工决策项 / 可继续推进项”语言；
5. 证据视图优先回答“当前是否可交付”，再提供审计细节。

## 本轮实施策略

0622 文档中 P0 已闭合，剩余大项多为 contract/worker/数据结构级 Epic。本轮不越界重做后端架构，而是优先实施一个低风险、高用户感知的前端 UX 闭环：

- `/loops/new` 创建入口增加提交就绪状态与禁用原因；
- 将请求长度、workspace 缺失、创建中状态转成明确的、可被屏幕阅读器感知的反馈；
- 用测试固定该行为，避免后续回归。
